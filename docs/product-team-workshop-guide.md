# Product Team Workshop Guide

This guide helps teams use the template matrix in a structured workshop.

## Workshop goal

Turn a vague memory feature into concrete design decisions: what is remembered, how users inspect it, how they correct it, and how they forget it.

## Participants

Include product design, product management, research, engineering, data or privacy stakeholders, and support or operations representatives when relevant.

## Suggested duration

Run a 90-minute workshop for one product flow or a half-day workshop for a broader memory strategy.

## Materials

- [Template Matrix Index](../templates/matrix-index.md)
- [Memory Risk Rubric](memory-risk-rubric.md)
- [Design Review Scorecard](design-review-scorecard.md)
- [Memory Lifecycle Rubric](memory-lifecycle-rubric.md)
- Current product flow or prototype

## Agenda

1. Define the user journey and memory moments.
2. List every memory object involved.
3. Classify each memory by source, scope, and risk.
4. Select templates for capture, inspection, correction, and forgetting.
5. Identify edge cases and trust risks.
6. Score the flow using the review scorecard.
7. Assign follow-up design decisions.
