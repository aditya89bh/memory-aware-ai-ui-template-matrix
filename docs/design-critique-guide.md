# Design Critique Guide

Use this guide to critique memory-aware product flows.

## Critique prompts

- What does the product remember?
- When does the user learn that memory exists?
- Where can the user inspect memory?
- How does memory affect the current output?
- Can the user correct memory from the moment of error?
- Can the user forget memory with clear consequences?
- Is memory scope visible?
- Is source or confidence visible when trust depends on it?

## Critique structure

1. Walk through the flow as a user.
2. Mark every memory capture and memory use moment.
3. Identify hidden or unclear memory behavior.
4. Map missing controls to templates.
5. Decide which issues block launch.

## Output

A useful critique should produce a short list of design changes, not only observations.
