# Release Checklist

Use this checklist before publishing a tagged release.

## Content

- README is current.
- Changelog includes the release version.
- Release notes are available.
- New links validate.
- No placeholder content remains.

## Quality

- Templates remain consistent with the schema.
- Examples include correction and forgetting moments.
- Worksheets are usable in reviews.
- Canvases include decision outputs.
- Pattern guides include risks and anti-patterns.

## Governance

- License is present.
- Code of conduct is present.
- Security policy is present.
- Citation metadata is present.
- Contribution guidance is present.
