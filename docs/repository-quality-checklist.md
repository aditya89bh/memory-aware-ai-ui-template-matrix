# Repository Quality Checklist

Use this checklist before publishing a release or accepting a major contribution.

## Content quality

- Terminology matches the taxonomy.
- Templates include problem, use case, UI pattern, controls, edge cases, risk, and example context.
- Examples include correction and forgetting moments.
- Worksheets are directly usable in a product or design review.
- Links are relative and point to existing files.

## Trust quality

- Memory capture is not silent for high-risk cases.
- Users have inspection, correction, and forgetting paths.
- Scope is visible where memory crosses boundaries.
- Source and confidence appear where outcomes depend on memory.
- Sensitive memory uses explicit consent and conservative defaults.

## Contribution quality

- Commit history is clean.
- No co-author trailers are present.
- New docs add practical value rather than repeating existing guidance.
- Public-facing docs explain the repository quickly.

## Phase 5 asset quality

- Visual frameworks include purpose, use case, diagram, application guidance, and common mistakes.
- Canvases include fillable fields, review questions, example entries, decision output, and facilitator notes.
- AI product pattern guides include context, memory objects, controls, risk, recommended templates, anti-patterns, and example moments.
- Navigation links from README, templates, and examples point to the new assets.
