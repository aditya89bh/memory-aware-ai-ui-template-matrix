# Release Notes v0.1

Version 0.1 establishes the repository as a practical reference for memory-aware product design.

## What is included

- A clear vision for memory as a first-class design material
- A taxonomy for memory objects, source, confidence, scope, lifecycle, actions, visibility, risk, conflicts, and outcomes
- A template matrix covering the core memory lifecycle
- Applied product flows across seven product categories
- Review tools for risk, lifecycle, stakeholder review, workshops, and critique
- Worksheets for hands-on product and design review
- Contribution and issue templates for community input

## Who should use this release

- Product designers defining memory interactions
- UX researchers evaluating trust and control
- Product teams reviewing memory-heavy features
- Consultants advising teams on user-facing memory systems
- Researchers exploring transparent and correctable memory experiences

## Recommended first use

Start with [How to Use This Repository in 30 Minutes](how-to-use-in-30-minutes.md), then apply the [Memory Flow Audit Worksheet](../worksheets/memory-flow-audit.md) to one product flow.
