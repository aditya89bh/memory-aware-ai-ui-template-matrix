# Glossary

This glossary defines the core terms used across the repository.

## Memory object

A discrete piece of information a product may reuse later, such as a preference, project constraint, source-backed fact, or correction.

## Source

The origin of a memory object. A source may be a direct user statement, correction, imported setting, project document, repeated behavior, or workspace record.

## Confidence

The product's level of certainty that a memory is accurate, current, and useful. Confidence should be visible when memory affects important outcomes.

## Scope

The boundary where memory applies, such as session, personal, project, workspace, or organization.

## Capture

The moment when the product proposes, confirms, imports, or infers memory.

## Inspection

The user's ability to see what the product remembers, where it came from, and how it is being used.

## Editing

A direct change to stored memory, such as revising wording, scope, or status.

## Correction

A repair flow for wrong, stale, or misleading memory that has affected product behavior.

## Forgetting

Removing, deactivating, archiving, expiring, or otherwise stopping memory from shaping future behavior.

## Provenance

The explanation of where memory came from and why it exists.

## Trust risk

The user-facing risk created when memory is wrong, hidden, stale, overbroad, sensitive, or difficult to control.
