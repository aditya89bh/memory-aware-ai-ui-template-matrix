# Research Methods for Memory-Aware Products

Use these methods to evaluate whether people understand, trust, and control memory behavior.

## Concept testing

Use early when deciding whether a memory feature should exist. Show users memory capture, inspection, correction, and forgetting concepts. Ask what feels helpful, surprising, or risky.

## Prototype walkthrough

Use clickable or static prototypes to test whether users notice memory indicators, understand scope, and find controls.

## Wrong-memory recovery test

Give participants an output shaped by incorrect memory. Observe whether they can identify the issue, correct the memory, and understand what changed.

## Scope comprehension test

Ask users where a memory will apply after capture: this chat, personal profile, project, workspace, or organization.

## Forgetting comprehension test

Ask users what they expect to happen after forget, archive, expiry, or source deletion actions.

## Longitudinal diary study

Use when memory changes over time. Ask participants to note when memory felt useful, wrong, stale, or invasive.

## Research outputs

Document findings as memory design requirements: required controls, confusing labels, missing recovery paths, and launch blockers.
