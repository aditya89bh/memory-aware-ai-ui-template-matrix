# How to Use This Repository in 30 Minutes

This guide helps a team get value from the repository in one short review session.

## 0-5 minutes: Pick one flow

Choose a product flow where memory affects an output, recommendation, automation, or user preference.

Write down:

- User goal
- Memory involved
- Where memory is captured
- Where memory affects the experience

## 5-10 minutes: Classify memory

Use the [Memory Risk Rubric](memory-risk-rubric.md) to classify each memory object as low, medium, high, or critical risk.

## 10-20 minutes: Select templates

Use the [Template Selection Guide](template-selection-guide.md) to choose templates for capture, inspection, correction, forgetting, scope, and provenance.

## 20-30 minutes: Audit the design

Use the [Memory Flow Audit Worksheet](../worksheets/memory-flow-audit.md) and [Design Review Scorecard](design-review-scorecard.md) to identify missing controls.

## Expected output

By the end, the team should know:

- Which memories are involved
- Which controls are missing
- Which templates to apply
- Which risks block launch
