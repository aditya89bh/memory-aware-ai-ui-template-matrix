# Repository Positioning

This repository is a design and UX strategy reference for memory-aware products.

## What it is

- A template matrix for memory-related product decisions
- A shared vocabulary for designers, researchers, product managers, and builders
- A practical review toolkit for trust, control, correction, and forgetting
- A source of examples for applied memory workflows

## What it is not

- A frontend component library
- A backend architecture guide
- A vendor comparison
- A privacy policy template
- A complete compliance framework

## Core claim

Memory is not only a technical capability. It is a user experience that must be designed across capture, inspection, editing, correction, forgetting, scope, provenance, and confidence.

## Audience promise

A reader should leave with practical ways to design memory that is transparent, controllable, and trustworthy.
