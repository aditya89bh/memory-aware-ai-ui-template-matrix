# Evaluation Metrics

Memory-aware products should be evaluated as user experiences, not only as storage or retrieval systems.

## Comprehension metrics

- Percentage of users who correctly identify when memory is being used
- Percentage who correctly explain memory scope
- Percentage who understand source and confidence labels

## Control metrics

- Success rate for editing memory
- Success rate for correcting wrong memory
- Success rate for forgetting single memory
- Success rate for changing scope

## Trust metrics

- Trust before and after correction
- Perceived control over memory
- Perceived clarity of forgetting
- Comfort with memory capture

## Quality metrics

- Rate of stale memory reports
- Rate of incorrect memory corrections
- Rate of rejected inferred memories
- Number of unresolved memory conflicts

## Operational metrics

- Time to resolve memory-related support reports
- Percentage of high-risk memories with review dates
- Percentage of shared memories with owner and source
- Completion of launch review checklist
