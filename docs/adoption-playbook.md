# Adoption Playbook

Use this playbook to introduce the repository into a product team's design process.

## Step 1: Start with one flow

Do not begin with an organization-wide memory strategy. Pick one flow where memory already affects user experience.

## Step 2: Run a memory inventory

Use the [Memory Flow Audit Worksheet](../worksheets/memory-flow-audit.md) to list memory objects, sources, scopes, and controls.

## Step 3: Classify risk

Use the [Memory Risk Rubric](memory-risk-rubric.md) to decide which controls are mandatory.

## Step 4: Select templates

Use the [Template Selection Guide](template-selection-guide.md) and [Pattern Combinations](pattern-combinations.md) to choose patterns.

## Step 5: Review with stakeholders

Use [Stakeholder Review Questions](stakeholder-review-questions.md) to align product, design, research, engineering, and governance concerns.

## Step 6: Add memory review to launch criteria

Use the [Design Review Scorecard](design-review-scorecard.md) before shipping memory-aware flows.
