# Usability Test Plan

This plan helps teams evaluate a memory-aware flow before launch.

## Study goal

Assess whether users can understand, inspect, correct, and forget product memory in a realistic workflow.

## Participants

Recruit people who match the product's target users and include participants with different privacy expectations and collaboration needs.

## Tasks

1. Identify what the product remembered.
2. Explain why an output was personalized.
3. Change a remembered preference.
4. Correct a wrong memory from an output.
5. Forget a specific memory.
6. Choose an appropriate scope for a new memory.

## Measures

- Task completion
- Time to find memory controls
- Correct interpretation of scope
- Confidence after correction
- Understanding of forgetting consequences
- Trust rating before and after the flow

## Moderator prompts

- What do you think the product remembers here?
- Where would you go to change this?
- What do you expect this forget action to do?
- Does this feel like enough control?

## Reporting format

Summarize findings by template family: capture, inspection, editing, correction, forgetting, scope, and provenance.
