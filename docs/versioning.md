# Versioning

This repository uses semantic versioning for public releases.

## Version format

`MAJOR.MINOR.PATCH`

## Major releases

Use a major release when the repository changes its structure, terminology, or template model in a way that affects how teams use it.

## Minor releases

Use a minor release when adding new templates, examples, worksheets, canvases, or pattern guides without breaking existing structure.

## Patch releases

Use a patch release for typo fixes, link fixes, clarification, or small improvements that do not change repository structure.

## Current release

`v1.0.0` represents the first stable public release of the repository.

## Release notes

Every release should update [CHANGELOG.md](../CHANGELOG.md) and include a concise summary of what changed, who it helps, and where to start.
