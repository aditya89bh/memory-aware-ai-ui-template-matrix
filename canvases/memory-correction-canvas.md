Plan how users identify, fix, verify, and recover from wrong memory.

## Review questions

- What does the user need to understand?
- What control must be available at the moment of need?
- What would make this memory feel surprising or unsafe?
- What source, scope, or confidence details are required?

## Example entries

- Product flow: onboarding preference setup
- Memory object: preferred summary length
- Scope: personal memory
- User controls: inspect, edit, forget, change scope

## Decision output

Document the selected templates, required controls, launch blockers, and owner for follow-up.

## Facilitator notes

Keep the discussion focused on user-facing behavior. If the group shifts into storage architecture, bring the conversation back to what the user sees, controls, and understands.
