# Memory Correction Templates

Memory correction templates help users repair wrong, misleading, or conflicting memory after the product has made an error or surfaced an inaccurate assumption.

Correction is a critical trust moment. The user is not only fixing data; they are testing whether the product can learn from mistakes in a controlled way.

## Design goal

Turn memory errors into clear, actionable repair flows.

## Templates in this family

- [That's Wrong Correction Flow](thats-wrong-correction-flow.md)
- [Conflicting Memory Resolver](conflicting-memory-resolver.md)
- [Correction Feedback Receipt](correction-feedback-receipt.md)

## Use this family when

- A product uses wrong memory in an output.
- A user disputes a remembered fact or preference.
- Two memories conflict.
- The product needs to show that correction changed future behavior.
- The correction affects multiple scopes or dependent outputs.

## Design cautions

Do not make users diagnose the system. Let them state what is wrong in plain language, then guide them through the memory repair.
