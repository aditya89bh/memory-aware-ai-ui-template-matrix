# Memory Editing Templates

Memory editing templates help users change stored memory before it causes repeated mistakes.

Editing is different from correction. Editing may be proactive and calm. Correction often happens after the product has already used memory incorrectly.

## Design goal

Give users direct control over memory wording, scope, status, and applicability.

## Templates in this family

- [Inline Memory Edit](inline-memory-edit.md)
- [Profile-Level Memory Manager](profile-level-memory-manager.md)
- [Temporary Session Override](temporary-session-override.md)

## Use this family when

- Users need to maintain preferences over time.
- Memory statements require better wording.
- Memory scope needs adjustment.
- Users need a temporary exception without deleting a durable preference.
- Product teams want memory control to be part of normal workflow.

## Design cautions

Editing should have clear consequences. If changing a memory affects future outputs, the product should confirm what will change and where the edit applies.
