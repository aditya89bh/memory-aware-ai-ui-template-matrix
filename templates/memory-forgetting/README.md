# Memory Forgetting Templates

Memory forgetting templates help users remove, deactivate, archive, or expire memory.

Forgetting is not only a privacy feature. It is also how users keep memory accurate, relevant, and emotionally acceptable.

## Design goal

Make memory reversible with clear consequences.

## Templates in this family

- [Forget Single Memory](forget-single-memory.md)
- [Forget by Category](forget-by-category.md)
- [Memory Expiry Control](memory-expiry-control.md)

## Use this family when

- Users need to delete a specific remembered detail.
- A category of memory is no longer welcome.
- Memory should expire after a task, project, or time period.
- The product handles sensitive or changing personal context.
- Users need confidence that memory is no longer applied.

## Design cautions

Explain the difference between deleting a memory, deleting its source, archiving it, and disabling its use. Users should not need to guess what "remove" means.
