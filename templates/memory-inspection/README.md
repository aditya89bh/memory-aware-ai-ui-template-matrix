# Memory Inspection Templates

Memory inspection templates help users see what a product remembers and understand how that memory affects the experience.

Inspection should not expose raw storage structures. It should translate memory into clear, reviewable statements with source, scope, and available controls.

## Design goal

Make memory understandable without making users manage a database.

## Templates in this family

- [Memory Drawer](memory-drawer.md)
- [Remembered Facts Timeline](remembered-facts-timeline.md)
- [Source-Linked Memory Card](source-linked-memory-card.md)

## Use this family when

- Users need to audit what the product remembers.
- Memory affects a recommendation, output, or automation.
- A product has multiple memory scopes.
- Users need confidence that corrections or deletions worked.
- Teams need visibility into shared workspace memory.

## Design cautions

Inspection should be layered. Start with summaries and allow deeper review when needed. A full memory list is useful only when it is searchable, filterable, and actionable.
