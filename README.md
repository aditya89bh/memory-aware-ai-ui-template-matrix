# Memory-Aware AI UI Template Matrix

A stable design reference for creating inspectable, editable, correctable, and forgettable memory experiences in AI products.

## Understand this repository in under 60 seconds

AI products increasingly remember preferences, projects, decisions, behavior, and context. If memory is hidden, users cannot understand, correct, or forget what shapes their experience.

This repository treats memory as a first-class design material. It gives teams templates, examples, rubrics, worksheets, canvases, visual frameworks, research tools, and product pattern guides for designing memory that is transparent, controllable, and trustworthy.

This is a design and UX strategy repository, not a frontend component library.

## Who this is for

- Product designers designing memory-aware flows
- UX researchers testing trust and control
- Product managers defining memory requirements
- AI application builders planning user-facing memory behavior
- Consultants running product reviews or workshops
- Researchers studying transparent and correctable memory systems

## Start here

1. Read [How to Use This Repository in 30 Minutes](docs/how-to-use-in-30-minutes.md).
2. Review the [Template Matrix Index](templates/matrix-index.md).
3. Pick an [Applied Example](examples/README.md) close to your product.
4. Audit your flow with the [Memory Flow Audit Worksheet](worksheets/memory-flow-audit.md).
5. Choose patterns with the [Template Selection Guide](docs/template-selection-guide.md).
6. Use [Canvases](canvases/README.md) or [Visual Frameworks](docs/visual-frameworks/README.md) to align the team.
7. Check launch readiness with the [Release Checklist](docs/release-checklist.md).
