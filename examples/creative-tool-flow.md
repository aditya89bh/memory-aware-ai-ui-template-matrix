# Creative Tool Flow

## Product context

A creative tool helps users draft campaigns, visual directions, scripts, and brand concepts. Memory can preserve style and constraints, but creative work often needs temporary departures from established preferences.

## User goal

A creative team wants the product to remember brand and project direction while allowing controlled experiments that do not overwrite durable creative memory.

## Memory involved

- Brand memory: confident but understated tone
- Project memory: avoid neon color palettes for current campaign
- Personal preference: show three concept directions before details
- Temporary override: explore a deliberately loud visual direction for one draft

## UI sequence

1. The team saves brand tone as workspace memory with source from brand guidelines.
2. Project constraints appear as memory chips beside generated concepts.
3. A temporary override lets the user explore a different direction for the current draft only.
4. The memory drawer separates brand, project, and temporary creative constraints.
5. Generated outputs include a short note showing which creative memories were used.

## Correction moment

A generated concept incorrectly applies the loud temporary override to all campaign drafts. The user marks the memory use wrong and confirms that the override applies only to the current draft.

## Forgetting moment

At campaign completion, the team archives project-specific visual constraints while keeping durable brand memory active.

## Trust risk

Creative memory can become restrictive if the product treats prior direction as permanent taste. Users need safe ways to experiment without polluting durable memory.

## Success criteria

- Brand, project, personal, and temporary memory are visually distinct.
- Users can override memory for creative exploration.
- Temporary experiments do not become durable memory accidentally.
- Archived campaign memory stops influencing new campaigns.
