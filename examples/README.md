# Examples

This directory contains practical examples showing how memory design templates can be applied to product scenarios.

Examples make the repository concrete. Each flow shows the user goal, memory involved, interface sequence, correction moment, forgetting moment, trust risk, success criteria, and relevant templates.

## Recommended example format

Each example should include:

- User goal
- Memory involved
- UI sequence
- Correction moment
- Forgetting moment
- Trust risk
- Success criteria
- Templates used

## Quality bar

Examples should be realistic and decision-oriented. They should include failure modes such as wrong memory, stale memory, sensitive memory, unclear source, and conflicts across scopes.
